## 一、平台定位与职责范围

### 1.1 平台背景

CMP（Cloud Management Platform）云管理平台，是搜狐内部基于 Panther 统一运维平台体系构建的云资源管理工具，旨在为各部门提供统一的云资源申请、审批、管理、授权入口，降低各业务部门自行操作云控制台的风险与门槛。

### 1.2 核心定位

> **CMP 是云资源的"申请-审批-授权-登记"中枢，而非云资源的"操控台"。**

CMP 平台的核心价值在于：

- **流程管控**：所有资源申请需经过审批流程，避免资源浪费和越权操作；
- **权限管理**：基于组织架构对资源访问权限进行精细化管控；
- **资产登记**：统一登记各部门云上资源，形成资产台账；
- **灵活配置**：平台支持按部门、按资源类型灵活配置审批流程和操作表单；
- **部门自治**：各部门可在授权范围内自主管理组织、资源和人员。

### 1.3 职责边界

#### ✅ CMP 负责的事项（一期）

| 类别    | 具体职责                                              |
| ----- | ------------------------------------------------- |
| 组织管理  | 维护部门 → 组的两级组织架构，管理用户、角色和权限                        |
| 云账号管理 | 管理各部门关联的阿里云主账号；管理个人持有的阿里云 RAM 子账号                 |
| 资源申请  | 用户通过平台提交资源申请（新建、变配、扩容、销毁等），经审批后由系统调用云 API 自动创建    |
| 资源台账  | 展示部门/组下的云资源列表，记录资源归属和权限信息                         |
| 资源授权  | 对具体资源进行 reporter / developer / master 三级权限的人员授权管理 |
| 资源包   | 支持将多个资源及其权限打包，批量授权给用户                             |
| 审批流程  | 对资源申请单进行审批（通过/驳回/撤回）；查看审批记录                       |
| 工单管理  | 提交和处理运维工单（不限于资源问题，可包含账号、网络、安全等类型）                 |
| 操作审计  | 记录平台内所有关键操作（含资源操作、权限变更、组织变更、云账号操作等）供管理人员查阅        |
| 部门配置  | 支持部门配置平台支持各资源类型的申请表单模板和审批流程模板                     |

#### ❌ CMP 不负责的事项（一期范围外）

复杂的云资源操控，应直接在云厂商控制台完成，举例如下：

| 不属于 CMP 的操作 | 说明 |
| --- | --- |
| 查看 Redis 监控指标 | 监控大盘请使用云厂商控制台或自建监控系统 |
| 配置 SLB/ALB/NLB 负载均衡规则 | 监听器、转发规则等配置在云控制台操作 |
| 查看 ECS 实例详细指标（CPU、内存、网络流量） | 监控指标通过云控制台或 Prometheus 查看 |
| 数据库 Schema 管理、SQL 执行 | 通过 DMS 或数据库客户端操作 |
| Kafka 消费者组管理、消息回溯 | 通过 Kafka 控制台操作 |
| VPC 路由表、NAT 规则配置 | 网络层精细配置在云控制台操作 |
| K8S 应用发布、Pod 管理 | 应用层操作通过 CI/CD 或 K8S Dashboard 完成 |

**一期总结**：CMP 聚焦于**资源的申请与授权管理**，强调**平台灵活配置**（资源类型、表单、审批流）和**部门各自灵活管理**（各部门独立配置自己的审批流、工单处理人、资源权限体系）。

### 1.4 二期规划

二期将在一期基础上，为各部门提供**云资源成本核算与分析**能力，包括：

- 按部门/组/项目维度的费用分摊
- 资源费用趋势分析、预算对比
- 资源利用率统计（CPU、内存使用率低的资源识别）
- 月度账单报告自动生成

## 三、功能详细说明

### 3.1 组织管理

#### 3.1.1 组织架构

组织架构是 CMP 权限体系的基础，采用**部门 → 组（一级组 → 二级组）**的两级结构。所有用户必须归属某个部门和组，资源也跟随组进行管理。

| 序号 | 功能模块 | 详细说明 | 原型截图 |
| --- | --- | --- | --- |
| 1 | 页面布局 | 采用左右分栏布局：左侧展示组织树（部门 → 组层级），右侧展示选中节点的详情和成员列表。顶部显示当前共有多少个部门。 | |
| 2 | 组织树 | 左侧树形结构展示所有部门及其下属组。每个部门可展开显示一级组，一级组下可有二级组。树节点可点击，点击后右侧展示对应详情和成员。支持多层嵌套显示。 | |
| 3 | 创建部门 | 点击「+ 创建部门」按钮（仅超级管理员可见）。弹窗需填写：部门名称（必填）、部门描述（选填）、部门负责人（必填，从现有用户中选择）。创建成功后树中新增部门节点。 | |
| 4 | 创建组 | 点击「+ 创建组」按钮（部门负责人及以上权限可见）。需填写：组名称（必填）、所属部门（必填）、父组（选填，支持创建二级组）、组描述（选填）、组长（选填，从部门成员中选择）。 | |
| 5 | 编辑部门/组 | 选中节点后，右侧详情区提供「编辑」入口。可修改名称、描述、负责人/组长等信息。部门负责人由超管指定；组长由部门负责人指定。 | |
| 6 | 删除部门/组 | 删除前系统弹出确认对话框，提示影响范围（如"该组下有 N 名成员，删除后成员将上浮至父级"）。若组下有资源，则资源转为"未归组资源"由部门负责人统一处理。 | |
| 7 | 添加成员 | 右侧成员列表区提供「+ 添加成员」按钮。弹窗展示当前部门内可选用户列表（搜索框支持按姓名/用户名筛选），选择用户后添加至当前组。同一用户只能属于一个组。 | |
| 8 | 移除成员 | 成员列表每行提供「移除」操作，弹出确认对话框后将该用户从本组移除。移除后用户仍属于该部门，但暂无归组（出现在未分配列表）。 | |
| 9 | 转移成员 | 支持将成员从当前组转移至同部门的其他组，选择目标组后确认。 | |
| 10 | 部门间转移 | 可将成员从当前部门转移至其他部门，需选择目标部门和目标组，转移后原部门权限失效。 | |
| 11 | 分配组（未归属成员） | 若用户尚未分配到任何组，在成员列表显示"未归组"标签，部门负责人可为其分配目标组。 | |
| 12 | 成员列表 | 右侧展示选中部门/组下的成员列表，含以下字段：姓名、用户名、手机号（脱敏）、角色标签、加入时间。支持搜索过滤。 | |

#### 3.1.2 角色管理

角色管理维护**功能模块权限**（即"能在平台做什么操作"）。注意：资源权限（reporter/developer/master）是另一套独立体系，不在此处管理。

| 序号 | 功能模块 | 详细说明 | 原型截图 |
| --- | --- | --- | --- |
| 1 | 角色列表 | 展示平台所有角色，包含字段：角色名称、角色类型（平台级/业务线级）、权限范围描述、用户数量、创建时间、操作按钮。内置角色不可删除，仅超管可创建新角色。 | |
| 2 | 内置角色 | 系统预置以下角色（不可删除）：**超级管理员**（平台级，所有功能全权限）、**运维负责人**（业务线级，资源目录/表单/流程管理）、**审核人**（业务线级，工单审核/资源申请审批）、**工单处理人**（业务线级，工单处理/转移）。业务线级角色仅对所在部门数据生效，实现部门数据隔离。 | |
| 3 | 创建角色 | 仅超级管理员可操作。填写：角色名称（必填）、角色类型（平台级/业务线级）、权限配置（按功能模块勾选具体权限点：查看/创建/编辑/删除/审批/处理/转移）。 | |
| 4 | 查看角色详情 | 点击角色行可展开查看：角色的完整权限矩阵（功能模块 × 权限点）、当前持有该角色的用户列表（姓名、用户名、所属部门）。 | |
| 5 | 权限模块说明 | 平台功能模块权限点定义：**组织架构管理**（查看/创建/编辑/删除）、**角色管理**（查看/创建/编辑/删除）、**云账号管理**（查看/创建/编辑/删除/审批）、**项目管理**（查看/创建/编辑/删除）、**资源管理**（查看/创建/编辑/删除/审批）、**工单管理**（查看/创建/审批/处理/转移）、**操作审计**（查看）。 | |
| 6 | 分配说明 | 超管可全局分配角色；部门负责人只能为本部门成员分配已有角色（不能创建新角色）。角色分配入口在用户管理和组织架构的成员详情中。 | |

#### 3.1.3 用户管理

管理平台所有用户账号。用户以**邮箱为唯一标识**，正常由 ERP 系统自动同步入库。

| 序号 | 功能模块 | 详细说明 | 原型截图 |
| --- | --- | --- | --- |
| 1 | 统计卡片 | 页面顶部展示用户数量统计：总用户数、已启用数、已禁用数。 | |
| 2 | 用户列表 | 表格字段包括：姓名、用户名、手机号（脱敏显示，如138\*\*\*\*1001）、所属部门、角色标签、状态（启用/禁用）、创建时间、最后登录时间、操作列（编辑/启用禁用/重置密码/分配角色）。 | |
| 3 | 搜索与筛选 | 支持按姓名/邮箱关键字搜索；支持按部门下拉筛选。 | |
| 4 | 创建用户 | 仅超级管理员可手动创建。填写：用户名（必填）、姓名（必填）、手机号（必填）、所属部门（必填）。系统使用 SSO 登录，无需设置密码。普通用户由 ERP 自动同步，无需手动创建。 | |
| 5 | 编辑用户 | 可修改：姓名、手机号、所属部门。用户名（邮箱前缀）不可修改，因其是系统唯一标识。 | |
| 6 | 启用/禁用 | 点击操作列「启用/禁用」按钮，弹出确认对话框（提示"禁用后该用户将无法登录平台"）。禁用状态的用户在列表中以灰色显示，最后登录时间保留记录。 | |
| 7 | 重置密码 | 对于非 SSO 特殊用户，超管可触发重置密码操作，弹出确认框后系统发送重置邮件。 | |
| 8 | 分配角色 | 超管或部门负责人点击「分配角色」，弹窗展示所有可用角色列表（带权限范围说明），支持多选，确认后角色生效。 | |
| 9 | 权限说明 | 超管可管理所有用户；部门负责人只能查看和管理本部门成员；普通成员只能查看用户列表（无法操作）。 | |

#### 3.1.4 部门配置

各部门在平台提供的默认配置基础上，进行部门级别的定制化覆盖。各部门配置互相独立，互不影响。

| 序号 | 功能模块 | 详细说明 | 原型截图 |
| --- | --- | --- | --- |
| 1 | 部门选择 | 页面顶部提供部门选择下拉框，切换后下方所有 Tab 内容切换至对应部门的配置。超管可查看和编辑所有部门；部门负责人/运维负责人只能查看本部门。 | |
| 2 | 主账号配置 Tab | 展示并管理当前部门关联的阿里云主账号。字段：当前关联账号名称、AccessKey ID（脱敏）、关联状态（已关联/未关联）、关联时间、关联操作人。提供「更换主账号」按钮。每个部门仅可关联一个主账号，用于平台代调用云 API 自动创建资源。 | |
| 3 | 资源操作配置 Tab | 展示本部门已开启支持的资源类型及其操作表单配置。列表字段：资源类型、操作类型（申请/变配/扩容/销毁等）、是否已定制（相对平台默认模板是否有字段覆盖）。点击「配置」可进入字段级别的定制，支持覆盖选项值（如将 VPC 选项限定为本部门已有的 VPC）。 | |
| 4 | 资源审批配置 Tab | 为本部门的各资源类型 × 操作类型组合配置审批流程。字段：资源类型、操作类型、审批流模板（可选：无审批/直属领导/直属领导+部门负责人/含指定审批人等）、指定审批人1、指定审批人2。各部门独立配置，互不影响；未配置的操作类型默认使用无审批流程。 | |
| 5 | 工单处理配置 Tab | 配置不同工单类别的默认处理人。字段：工单类别（账号权限类/资源问题类/网络问题类/安全合规类/其他）、默认处理人（从本部门工单处理人角色中选择）。用户提交工单时，系统根据此配置自动指派处理人。 | |

### 3.2 账号 & 项目

#### 3.2.1 云账号

云账号分为两类：**主账号**（部门级，供平台自动调用 API）和**子账号**（个人级 RAM 账号，供人工在云控制台操作）。

**主账号管理 Tab**

| 序号 | 功能模块 | 详细说明 | 原型截图 |
| --- | --- | --- | --- |
| 1 | 主账号列表 | 展示所有部门的主账号关联情况。字段：部门名称、云厂商（一期固定为阿里云）、主账号名称（含 AccessKey 脱敏 ID）、关联操作人、关联时间、状态（正常/未关联）。 | |
| 2 | 说明信息 | 页面顶部蓝色提示：**每个部门仅可关联一个阿里云主账号，由部门负责人操作。平台使用该账号统一申请云上资源、调用云上接口。关联的主账号必须具备创建子账号的权限。** | |
| 3 | 关联主账号 | 点击未关联部门的「关联」操作，弹窗填写：云厂商（阿里云）、AccessKey ID、AccessKey Secret（加密存储）、账号别名。提交后平台验证账号可用性，验证通过后完成关联。 | |
| 4 | 查看详情 | 已关联状态的主账号可查看详情：账号名称、关联时间、关联人、账号状态（平台定期验证可用性）。不展示明文 AK/SK。 | |
| 5 | 确认操作 | 关键操作（解绑/更换主账号）需二次确认，弹出确认框说明影响范围（解绑后本部门资源申请将无法执行）。 | |

**子账号管理 Tab**

| 序号 | 功能模块 | 详细说明 | 原型截图 |
| --- | --- | --- | --- |
| 1 | 数据范围说明 | 黄色提示：**仅展示我的子账号和本部门子账号（部门负责人视角）。子账号仅归属于个人，仅供本人查看。** 橙色警告：**通过子账号创建的云上资源属于孤儿资源，需由部门负责人进行分配。** | |
| 2 | 子账号列表 | 字段：子账号名称（如 wanghr-dev）、云厂商、RAM 用户名（脱敏）、绑定权限包、有效期类型（长期持有/临时有效+天数）、创建时间、状态（正常/审批中/已回收）、操作（查看凭证/申请续期/回收）。 | |
| 3 | 搜索筛选 | 支持按部门（部门负责人视角可选）、状态（正常/审批中/已回收）筛选。 | |
| 4 | 申请子账号 | 点击「+ 申请子账号」，填写：账号名称（如 yourname-dev，必填）、权限包（从预配置权限包列表中选择）、有效期类型（长期持有/临时有效，临时则需选择 7/14/30/90 天）、用途说明（必填）。提交后进入审批流程，审批通过后系统自动在部门主账号下创建 RAM 子账号并绑定相应策略。 | |
| 5 | 查看凭证 | 子账号状态为"正常"时，持有人可查看登录 URL、RAM 用户名、AccessKey ID、AccessKey Secret（敏感信息，仅本人和部门负责人可见，需二次鉴权）。 | |
| 6 | 临时账号续期/回收 | 临时子账号到期前，持有人可申请续期；部门负责人可随时回收（即吊销该 RAM 账号的权限并标记为已回收）。 | |

#### 3.2.2 项目管理

| 2 | 标签页切换 | 提供三个视角的 Tab：**总览**（根据权限展示可见范围内全部工单）、**我创建的**（仅显示本人创建的工单）、**我处理的**（仅工单处理人/部门管理员可见，展示指派给自己的工单）。 | |
| 3 | 搜索筛选 | 支持：关键字搜索（工单号/标题）、工单类别筛选（账号权限类/资源问题类/网络问题类/安全合规类/其他）、状态筛选（待处理/处理中/已完结）、部门筛选（管理员视角）。 | |
| 4 | 工单列表表格 | 字段：工单号（如 TK-20260316-001）、标题、工单类别、状态（带颜色：待处理黄色/处理中蓝色/已完结绿色）、申请人（含所属部门）、处理人、创建时间、最后更新时间、操作（查看详情/处理/转移）。 | |
| 5 | 创建工单 | 点击「+ 创建工单」，填写：工单标题（必填）、工单类别（必选：账号权限类/资源问题类/网络问题类/安全合规类/其他）、问题描述（必填，富文本区域）、关联资源（选填，可选择具体资源）、期望完成时间（选填）。提交后系统根据部门工单配置自动指派处理人，状态变为"待处理"。 | |

| 7 | 处理工单 | 工单处理人点击「处理」，以侧边抽屉方式打开处理面板。面板展示工单详情（只读），底部提供：处理进展（文本输入）、状态更新（继续处理中/标记已完结）。处理记录追加到时间线。 | |
| 8 | 工单转移 | 处理人可将工单转移给本部门的其他处理人，填写转移原因，确认后被转移人接收工单并收到通知。 | |
| 9 | 数据隔离 | 工单数据按部门隔离：普通用户只能看到自己创建的工单；工单处理人能看到指派给自己的工单；部门负责人能看到本部门所有工单；超管能看到全平台工单。 | |
| 10 | 工单类别说明 | **账号权限类**：子账号登录异常、RAM 策略配置、权限申请等；**资源问题类**：ECS/RDS/Redis 等资源故障、性能问题；**网络问题类**：VPC 连通性、SLB 配置、DNS 问题；**安全合规类**：安全组配置、权限越权、合规审查；**其他**：技术咨询、文档需求等。 | |

#### 3.5.2 操作审计

操作审计记录平台内所有关键操作的日志，用于事后追溯和合规审查。

| 序号 | 功能模块 | 详细说明 | 原型截图 |
| --- | --- | --- | --- |
| 1 | 权限说明 | 蓝色提示：**审计日志仅对管理角色可见（超管、部门负责人、运维负责人、工单处理人），按部门做数据隔离。** | |
| 2 | 审计日志列表 | 字段：操作时间、操作人、所属部门、操作类型（带颜色标签）、操作对象（资源名称/用户名等）、操作描述、来源 IP、变更前、变更后。 | |
| 3 | 搜索筛选 | 支持：关键字搜索（操作人/操作对象）、操作类型筛选（资源操作/权限变更/组织架构变更/云账号操作/审核流程操作/工单操作/账号管理）、部门筛选、时间范围选择（开始日期至结束日期）。 | |

| 6 | 导出日志 | 提供「导出日志」按钮，按当前筛选条件导出 Excel/CSV 格式日志，用于合规报告。 | |

## 四、权限体系说明

### 4.1 两套独立权限体系

CMP 存在两套并行的权限体系，相互独立：

| 权限体系 | 说明 | 维护位置 |
| --- | --- | --- |
| **功能模块权限**（角色） | 控制用户能在平台做哪些操作（如创建资源、审批工单等）。通过"角色管理"定义和分配。 | 角色管理页面 |
| **资源访问权限** | 控制用户对具体资源的访问级别（reporter/developer/master）。通过资源列表中的"授权"操作分配，或通过资源包批量分配。 | 资源列表、资源包 |

### 4.2 资源权限级别说明

| 权限级别 | 说明 | 持有限制 |
| --- | --- | --- |
| **reporter** | 只读查看：可查看资源基础信息、运行状态 | 无限制 |
| **developer** | 读写操作：可执行数据库账号创建、查询等操作 | 无限制 |
| **master** | 完全管理：可对资源进行所有操作，并可为他人授权 | 数据库/中间件类资源的 master 仅组长和部门负责人可持有 |

### 4.3 平台角色与数据隔离

| 角色 | 数据可见范围 | 特殊权限 |
| --- | --- | --- |
| 超级管理员 | 全平台所有数据 | 创建部门、创建角色、管理所有用户 |
| 部门负责人 | 本部门所有数据 | 管理本部门成员角色、处理未归组资源、配置部门审批流和工单处理人 |
| 运维负责人 | 本部门资源管理相关数据 | 配置资源目录、表单模板、审批流程 |
| 审核人 | 本部门审批相关数据 | 审核资源申请单 |
| 工单处理人 | 本部门工单数据 | 处理和转移工单 |
| 普通成员 | 自己创建/持有的数据 | 申请资源、提交工单、查看申请记录 |

## 五、关键业务流程

### 5.1 资源申请流程

```
用户（资源列表页点击"申请资源"）
  → 选择资源类型 & 填写申请表单（含所属项目、付费配置）
  → 系统根据"部门配置-资源审批配置"确定审批流
  → 提交申请单 → 进入"申请记录"可查看进度
  → 审批节点依次流转（直属领导 → 部门负责人 → 指定审批人）
  → 全部通过：系统调用部门主账号的云 API 自动创建资源
    → 资源出现在"资源列表"，申请人默认获得 master 权限（视资源类型）
  → 任一驳回：申请单变为"已驳回"，流程终止，资源不创建
```

### 5.2 子账号申请流程

```
用户（云账号-子账号管理页点击"申请子账号"）
  → 填写：账号名称、权限包、有效期类型、用途说明
  → 提交 → 进入审批流（直属领导 → 部门负责人）
  → 审批通过：系统在部门主账号下自动创建 RAM 子账号，绑定权限包中的 RAM 策略
  → 持有人可在"云账号-子账号"页面查看凭证信息（登录URL、AK/SK）
```

### 5.3 未归组资源处理流程

```
云上子账号直接创建资源 OR 组解散导致资源无归属
  → 平台定期同步云账号下的资源列表，发现未在平台管理的资源
  → 资源出现在"未归组资源"列表，标注来源
  → 部门负责人在"未归组资源"页面为其分配目标组
  → 资源纳入平台管理，归属到指定组
```

## 六、非功能性要求

| 方面 | 要求 |
| --- | --- |
| 权限安全 | AK/SK 等敏感凭证必须加密存储，展示时默认脱敏，查看明文需二次鉴权 |
| 数据隔离 | 各部门数据严格隔离，服务端接口层面实施，不依赖前端展示控制 |
| 审计完整性 | 所有关键操作均记录审计日志，不可删除和篡改 |
| 云 API 异步 | 云资源创建/变配等操作为异步，系统需轮询或 Webhook 监听结果并更新资源状态 |
| 错误处理 | 云 API 调用失败时（如 AK 过期、配额不足），需将错误信息反馈至申请单，并通知申请人 |
| SSO 集成 | 平台用户登录通过 Panther 统一 SSO 完成，CMP 不管理用户密码 |
| ERP 同步 | 用户账号正常由 ERP 系统定期同步，超管可手动创建特殊用户 |
