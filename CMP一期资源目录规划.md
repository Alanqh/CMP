# CMP一期资源目录规划

## 一、规划原则

1. **覆盖高频刚需**：优先纳入员工日常申请频率最高、业务依赖度最强的资源类型
2. **资源形态分类**：按「实例型」「集群型」「二次资源」三类分别设计表单与流程
3. **基础设施托管**：网络底座（VPC/交换机/安全组）、安全产品由运维统一管理，不开放自助申请
4. **一期聚焦**：优先做好计算、数据库、中间件、容器等核心链路，大数据/AI/安全类资源纳入二期

## 二、资源形态定义

| 形态 | 说明 | 典型特征 | 示例 |
|------|------|----------|------|
| **实例型** | 以单个实例为最小管理单元，独立申请、独立计费 | 一次申请一个实例，实例间相互独立 | ECS、Redis、RDS |
| **集群型** | 以集群为管理单元，集群内包含多个节点/组件 | 一次申请一个集群，集群内资源统一管理 | ACK、Kafka、Lindorm |
| **二次资源** | 依附于已有实例/集群的子资源，不可独立存在 | 需先有父资源，在其上创建子资源 | Kafka Topic、RDS 数据库/账号、ACK Namespace |

## 三、一期资源目录

### 3.1 计算与容器

| 资源类型 | 资源形态 | 优先级 | 纳入理由 |
|----------|----------|--------|----------|
| **ECS 云服务器** | 实例型 | P0 | 最基础的计算资源，几乎所有业务线都需要 |
| **ACK 容器服务（K8s）** | 集群型 | P0 | 容器化部署主力平台，业务广泛依赖 |
| **ACR 容器镜像服务** | 实例型 | P1 | 容器镜像管理，与 ACK 配套使用 |

> **ACK 说明**：一期集群由运维创建和管理，面向开发者开放 **Namespace 级别**的二次申请（指定资源配额 CPU/内存 Limits），不开放集群级操作。

**ECS 申请核心参数**：地域/可用区、实例规格族、vCPU/内存、系统盘类型及大小、数据盘、镜像、VPC/交换机、安全组、计费方式、所属项目

**ACR 申请核心参数**：地域、实例类型（个人版/企业版）、命名空间、仓库名称、所属项目

### 3.2 数据库

| 资源类型 | 资源形态 | 优先级 | 纳入理由 |
|----------|----------|--------|----------|
| **PolarDB** | 实例型 | P0 | 核心关系型数据库，业务主力存储 |
| **RDS PostgreSQL** | 实例型 | P0 | 部分业务线依赖 PG 生态 |
| **Redis** | 实例型 | P0 | 缓存场景刚需，业务普遍依赖 |
| **MongoDB** | 实例型 | P1 | 文档型存储，多个业务线在用 |
| **Lindorm** | 集群型 | P1 | 多模数据库，覆盖时序/宽表/搜索场景 |

**PolarDB 申请核心参数**：地域/可用区、数据库引擎（MySQL/PG）及版本、节点规格、节点数量、存储空间、网络类型（VPC）、计费方式、所属项目

**Redis 申请核心参数**：地域/可用区、架构类型（标准版/集群版/读写分离版）、实例规格（内存容量）、副本数、网络类型（VPC）、计费方式、所属项目

**数据库二次申请**：在已有 PolarDB/RDS 实例上申请 Database 和账号，走独立审核流程

### 3.3 消息队列与中间件

| 资源类型 | 资源形态 | 优先级 | 纳入理由 |
|----------|----------|--------|----------|
| **Kafka** | 实例型 | P0 | 异步通信核心组件，业务线普遍使用 |
| **RocketMQ** | 实例型 | P1 | 事务消息/顺序消息场景，部分业务依赖 |
| **RabbitMQ** | 实例型 | P2 | 使用范围较窄，但已有业务在用 |
| **Nacos** | 实例型 | P0 | 服务注册与配置中心，微服务架构基础设施 |
| **XXL-Job** | 实例型 | P1 | 分布式任务调度，多个业务线使用 |

**Kafka 申请核心参数**：地域/可用区、实例规格（流量峰值）、磁盘类型及大小、网络类型（VPC）、计费方式、所属项目

**Kafka 二次申请**：在已有 Kafka 实例上申请 Topic，需指定分区数、副本数、消息保留时长，走独立审核流程

**RocketMQ 申请核心参数**：地域/可用区、实例规格、消息存储时长、网络类型（VPC）、计费方式、所属项目

### 3.4 网络与流量

| 资源类型 | 资源形态 | 优先级 | 纳入理由 |
|----------|----------|--------|----------|
| **LB 负载均衡** | 实例型 | P0 | 服务暴露和流量分发的基础设施 |
| **NAT 网关** | 实例型 | P1 | VPC 内实例访问公网的通道 |
| **弹性公网 IP（EIP）** | 实例型 | P1 | 公网访问入口，与 ECS/NAT/LB 绑定使用 |
| **云原生 API 网关** | 实例型 | P1 | API 管理与流量治理，微服务架构核心组件 |
| **CDN** | 实例型（域名维度） | P2 | 静态资源加速，使用场景相对集中 |

**LB 申请核心参数**：地域/可用区、产品类型（CLB/ALB/NLB）、实例规格、网络类型（公网/私网）、计费方式、所属项目

**EIP 申请核心参数**：地域、带宽峰值、计费方式（按流量/按带宽）、所属项目

> **说明**：VPC、交换机、安全组、云解析 DNS 为网络底座，一期由运维统一规划，不开放自助申请。

### 3.5 存储

| 资源类型 | 资源形态 | 优先级 | 纳入理由 |
|----------|----------|--------|----------|
| **OSS 对象存储** | 实例型（Bucket） | P0 | 静态资源、文件、备份等存储场景广泛使用 |
| **NAS 文件存储** | 实例型 | P2 | 共享文件存储，容器/计算场景有需求 |

**OSS 申请核心参数**：地域、Bucket 名称、存储类型（标准/低频/归档）、读写权限（私有/公共读）、所属项目

### 3.6 数据管理与运维工具

| 资源类型 | 资源形态 | 优先级 | 纳入理由 |
|----------|----------|--------|----------|
| **DMS 数据管理** | 平台型 | P0 | 数据库变更/查询统一入口，安全管控刚需 |
| **DTS 数据传输** | 任务型 | P1 | 数据迁移/同步/订阅，数据库运维高频操作 |
| **SLS 日志服务** | 实例型（Project） | P1 | 日志采集/分析/告警，可观测体系核心 |
| **ARMS 可观测** | 平台型 | P1 | APM/链路追踪/Prometheus 监控，可观测体系核心 |

> **说明**：DMS/ARMS 为平台型产品，一期以「开通/授权」为主要管理动作，无需走实例申请流程。

## 五、资源权限模型

基于 `master / developer / reporter` 三级权限体系，按资源大类细化：

| 权限等级 | 计算（ECS/ACK） | 数据库（PolarDB/Redis/MongoDB） | 中间件（Kafka/RocketMQ/Nacos） | 网络（LB/NAT/EIP） | 存储（OSS/NAS） |
|----------|----------------|-------------------------------|-------------------------------|-------------------|----------------|
| **master** | 变配、销毁、重启、登录、授权 | 变配、销毁、账号管理、授权 | 变配、销毁、Topic/Group 管理、授权 | 变配、销毁、绑定/解绑、授权 | 读写、删除 Bucket、授权 |
| **developer** | 重启、登录、查看 | 数据读写、查看 | Topic 读写（生产/消费）、查看 | 查看 | 读写、查看 |
| **reporter** | 仅查看监控/配置 | 仅查看配置/慢日志 | 仅查看 Topic 列表/消费组 | 仅查看 | 仅读、查看 |

## 六、一期不纳入的资源

以下资源暂不纳入一期，按类型说明原因：

### 大数据与AI类

| 资源类型 | 不纳入原因 |
|----------|-----------|
| MaxCompute | 大数据计算引擎，申请模型与常规资源差异大，需单独设计 |
| Hologres | 实时数仓，与 MaxCompute 配套，同批次规划 |
| DataWorks | 数据开发平台，平台型产品，管理逻辑独立 |
| Flink | 实时计算，任务型资源，需独立设计作业管理流程 |
| GPU | 算力资源稀缺且昂贵，需独立的审批和配额管控 |
| AI 网关 | 新兴产品，使用场景尚在探索 |

### 安全类

| 资源类型 | 不纳入原因 |
|----------|-----------|
| 堡垒机 | 安全基础设施，由安全团队统一管理 |
| 云防火墙 | 安全策略集中管控，不适合自助申请 |
| 主机及容器安全 | 安全 Agent 类产品，统一部署而非按需申请 |
| 安全产品（WAF 等） | 安全团队统一运营 |

### 基础设施类

| 资源类型 | 不纳入原因 |
|----------|-----------|
| 云解析 DNS | 域名管理，运维统一管控 |
| PrivateLink（私有连接） | 网络底座，运维统一规划 |
| ESA（动态加速） | 使用场景有限，可后续按需接入 |
| ZooKeeper | 逐步被 Nacos 替代，不鼓励新增 |
| HBase | 逐步被 Lindorm 替代，不鼓励新增 |

### 其他

| 资源类型 | 不纳入原因 |
|----------|-----------|
| CMS 云监控 | 基础监控能力已由 ARMS/SLS 覆盖 |
| DAS 数据库自治 | 辅助工具，非独立资源申请场景 |
| LTS | 功能与 SLS 重叠，统一归口 SLS |
