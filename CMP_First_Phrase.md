# CMP平台一期建设方案

## 一、组织架构管理

### 1.1 架构创建与人员归属
1. **部门创建**：仅超级管理员可创建部门，创建时必须指定部门负责人
2. **组创建**：部门负责人可创建一级组和二级组，创建时必须指定组长
3. **人员归属规则**：
   - 每个用户仅能归属一个部门
   - 每个用户仅能加入一个组（不可同时加入多个组）
   - 组织架构层级：部门 → 一级组 → 二级组 → 成员

### 1.2 组织架构的资源权限穿透规则
1. **上下级权限**：
   - 部门负责人拥有本部门下所有资源的 `master` 权限
   - 各级组长拥有本组下所有资源的 `master` 权限
2. **同组权限**：组内普通成员对本组下所有资源默认拥有 `developer` 权限
3. **权限体系说明**：
   - 功能模块权限（角色）与资源权限（reporter/developer/master）是两套独立的权限体系
   - 资源所属功能模块默认对所有用户开放访问入口
   - 角色仅控制功能模块内的管理操作权限
4. **多权限合并规则**：同一用户通过「组织架构穿透、角色授权、资源直接授权」获得的资源权限取并集

### 1.3 人员变动权限规则
1. 资源为组级资产，人员调组、离职均不影响资源原有归属
2. 人员调组后：
   - 自动失去原组所有资源的 developer 权限
   - 按新组架构获得新组资源的 developer 权限
   - 工单处理人角色权限随人员调组同步转移，原部门/组的工单处理权限自动回收

### 1.4 组解散处理
组解散时，组下所有资源自动上浮至部门层级，由部门负责人统一持有该部分资源的 `master` 权限

## 二、用户管理

### 2.1 用户创建与同步
1. **用户来源**：用户正常由 ERP 系统自动同步，以邮箱为唯一标识
2. **手动创建**：仅超级管理员可手动创建特殊用户
3. **登录方式**：系统使用 SSO 单点登录，无需管理密码

### 2.2 用户状态管理
1. 用户状态包括：正常、禁用
2. 超级管理员可禁用/启用用户
3. 禁用用户后，该用户无法登录系统，但不影响其已创建的资源归属

## 三、角色管理

### 3.1 角色分类与权限范围
一期仅设置两类核心角色，简化权限体系：
1. **平台管理角色 - 超级管理员**：
   - 拥有全平台所有功能模块、所有部门数据的所有操作权限
   - 为最高权限角色
2. **部门管理角色**：
   - 由超级管理员预制创建
   - 仅拥有所在部门内对应功能模块的数据操作权限，无跨部门权限
   - 包含：部门负责人、运维负责人、审核人、工单处理人等细分角色

### 3.2 角色权限配置
1. 平台按功能模块抽象核心操作权限，由研发提前封装权限接口/选项
2. 角色权限为功能模块层面权限，与资源的 reporter/developer/master 权限体系相互独立
3. 权限配置包括：
   - 组织架构管理权限
   - 用户管理权限
   - 角色管理权限
   - 云账号管理权限
   - 项目管理权限
   - 资源管理权限
   - 工单管理权限
   - 审计日志查看权限

### 3.3 角色创建与分配
1. **创建主体**：仅超级管理员可创建角色，并为角色配置对应的功能模块操作权限
2. **分配主体**：部门负责人可在超级管理员创建的角色范围内，为部门内成员指定已存在的角色
3. 一期不开放用户自定义角色功能

## 四、云账号管理

### 4.1 主账号管理
1. **绑定规则**：
   - 由部门负责人将企业云账号关联至本部门
   - 一个部门仅能绑定一个主账号
   - 主账号作为平台对接云上资源的核心账号
2. **使用方式**：
   - 平台申请云上资源、调用云上接口时，统一使用部门主账号操作
   - 一期暂不采用子账号对接云上接口的双层拦截方案
   - 由平台侧做好权限拦截控制
3. **权限要求**：
   - 关联的主账号必须具备云上创建子账号的权限
   - 必须支持云上资源同步能力（VPC、安全组等预置网络资源）

### 4.2 子账号管理
1. **申请与创建**：
   - 普通成员可自主申请子账号
   - 平台提供预置权限模板（按粗粒度打包云上权限）
   - 用户选择模板后走自动化创建流程
   - 特殊复杂权限需求可人工介入配置
2. **权限模板**：
   - K8s集群管理权限
   - ECS操作权限
   - 管理员权限（仅限部门负责人申请）
   - MySQL管理权限
   - 其他预置权限模板
3. **归属与查看**：
   - 子账号归属于申请人个人
   - 仅申请人可查看自己的子账号
   - 部门负责人可查看本部门所有子账号
4. **资源归属**：通过子账号创建的云上资源属于孤儿资源，需要由部门负责人或资源管理员进行分配
5. **生命周期管理**：
   - 支持长期持有和临时有效两种类型
   - 申请人在申请时自主选择
   - 部门负责人可对本部门子账号进行回收、销毁操作
6. **存量子账号处理**：部门主账号关联平台时，若已存在存量子账号，默认归属给部门负责人

### 4.3 可用区列表
1. 展示主账号下的可用区资源（VPC、交换机、安全组等）
2. 支持手动刷新同步云上最新资源
3. 按主账号维度展示，切换账号后自动刷新列表
4. 显示最后同步时间

## 五、项目管理

### 5.1 项目创建与使用
1. **创建主体**：由部门负责人统一创建本部门的项目，一期不开放普通成员创建权限
2. **使用规则**：
   - 资源申请时必须为资源指定一个项目
   - 一个资源仅能绑定一个项目，不可多绑
   - 与云上资源唯一归属规则对齐

### 5.2 项目核心功能
1. 作为资源的逻辑分组容器，组织和管理同类/同项目资源
2. 实现资源的批量管理、批量授权
3. 成本核算能力规划至二期建设，一期项目仅作为资源归属的逻辑维度

### 5.3 项目维度资源查看
1. 部门内用户可查看本部门所有项目列表
2. 仅能查看到指定项目下自己拥有权限的资源，无权限资源不可见
3. 共享资源通过项目标记后，可通过资源授权功能开放给其他组/成员

### 5.4 项目统计
项目列表展示每个项目的资源数量统计，便于部门负责人了解项目资源使用情况

## 六、资源生命周期管理

### 6.1 资源目录配置
1. **配置主体**：由平台运维同学统一配置全平台资源目录，标准化资源分类
2. **资源分类**（一期参考云上官方分类）：
   - **计算类**：ECS云服务器（含云硬盘、弹性IP等强绑定资源）、K8S集群
   - **数据库类**：RDS MySQL、PostgreSQL、MongoDB、Redis
   - **网络与负载均衡类**：SLB、ALB、NLB、云原生网关
   - **中间件类**：Kafka、PolarDB
   - **大数据与搜索分析类**：Elasticsearch、MaxCompute、Flink、实时数仓
   - **存储类**：对象存储OSS、块与文件存储、CDN流量包
   - **网络基础类**：VPC、NAT网关、交换机、安全组（仅做资源同步，不开放申请流程）
3. **资源目录展示**：
   - 资源目录从后端接口同步
   - 展示平台支持的阿里云资源类型及其元数据结构
   - 分类、支持操作、审批操作等由后端统一维护

### 6.2 资源申请流程
1. **申请入口**：用户点击"创建资源"按钮，选择资源类型后进入申请表单
2. **表单填写**：
   - 必填��：资源名称、所属项目、所属组、资源规格
   - 可选项：根据资源类型不同而不同（如ECS需选择操作系统、VPC配置等）
   - 表单字段遵循"平台预置基础字段 + 部门定制固定字段 + 用户极简填写字段"三层设计
3. **审核流程**：
   - 基于ERP低代码平台统一配置
   - 一期采用固定流程模板，按资源类型、操作类型预置审核节点
   - 审核节点默认按组织架构上下级设置（直属组长 → 部门负责人 → 运维负责人）
   - 支持部门负责人指定部分节点审核人
4. **自动创建**：审核通过后，系统使用部门主账号自动创建云上资源

### 6.3 资源操作类型
1. **申请**：创建新资源
2. **变配**：修改资源规格（如ECS升降配）
3. **销毁**：删除资源
4. **扩容**：扩展资源容量（如磁盘扩容）
5. **单独申请附属资源**：如云硬盘、弹性IP等，审核通过后自动挂载至对应ECS实例
6. **二次申请**：如Kafka Topic、ES索引等，在已有资源基础上申请子资源

### 6.4 资源归属与权限
1. **资源归属**：
   - 资源申请成功后，默认归属到申请人所在的组，为组级资产
   - 不因人员变动改变归属
   - 通过主账号同步的云上预置资源（如VPC、安全组），默认归属到部门层级
   - 部门负责人持有部门级资源的 master 权限
2. **资源权限分类**（与组织架构权限联动，为资源层面专属权限）：
   - **reporter**：只读查看权限
   - **developer**：读写操作权限（含数据库账号创建、Kafka Topic申请等）
   - **master**：完全授权管理权限
3. **权限分配规则**：
   - 组长拥有本组资源的 master 权限
   - 组员拥有本组资源的 developer 权限
   - 跨组授权时可授予 reporter 或 developer 权限
   - 数据库、中��件等核心资源，master 权限仅对组长/部门负责人开放
4. **权限说明展示**：资源列表页面顶部展示权限说明，帮助用户理解权限体系

### 6.5 资源列表与筛选
1. **分类展示**：按资源类型分Tab展示（计算资源、数据库、网络、中间件、大数据、存储）
2. **筛选条件**：
   - 搜索：资源名称/ID
   - 资源类型：下拉选择具体资源类型
   - 所属组：筛选特定组的资源
   - 所属项目：筛选特定项目的资源
3. **权限过滤**：用户仅能看到自己拥有权限的资源

### 6.6 资源详情
1. 点击资源可查看详细信息
2. 展示资源基本信息、配置信息、权限信息
3. 展示关联的工单记录
4. 支持资源操作（根据权限显示可用操作）

### 6.7 孤儿资源管理
1. **孤儿资源来源**：
   - 通过子账号在云上直接创建但未被平台管理的资源
   - 组解散后上浮的资源
   - 因其他原因脱离组归属的资源
2. **管理主体**：由部门负责人统一分配到指定组
3. **分配操作**：
   - 支持单个资源分配
   - 支持批量分配
   - 分配时选择目标组
4. **筛选功能**：
   - 搜索资源名称/ID
   - 按资源类型筛选

### 6.8 资源申请表单模板配置
1. **配置主体**：由各部门运维同学自主维护本部门的资源申请表单模板
2. **配置入口**：部门配置 → 资源操作配置
3. **表单设计原则**：
   - 平台预置基础字段（如地域、可用区、规格）
   - 部门运维可固定部分非核心字段（如操作系统版本、VPC配置）
   - 最终用户仅需填写少量关键字段（如资源名称、规格选择）
4. **模板管理**：
   - 不同资源类型单独配置表单模板
   - 公共字段（如地域、所属项目、所属组）做抽象复用

## 七、工单系统管理

### 7.1 工单核心定位
工单系统为CMP平台一期核心模块，资源申请审核流程为工单系统的核心子类，同时覆盖资源操作、问题反馈、权限申请等业务场景，实现业务流程全链路留痕与运维工作量化考核。

### 7.2 工单类型（一期固定，不开放自定义）
1. **资源类工单**：资源申请、变配、销毁、扩容、二次申请（Kafka Topic/ES索引）、资源包申请/续费
2. **权限类工单**：子账号申请、资源权限授权/回收、角色分配
3. **操作类工单**：资源重启、迁移、孤儿资源分配
4. **服务类工单**：问题反馈、bug提交、数据库账号创建/变更

### 7.3 工单创建与处理
1. **创建主体**：平台所有用户均可创建工单，按工单类型选择对应模板
2. **处理主体**：由部门负责人指定的工单处理人处理
3. **处理流程**：
   - 工单创建 → 审核人审核 → 工单处理人执行 → 工单完结
   - 与ERP低代码平台审核流程联动
   - 审核节点复用组织架构上下级规则
4. **工单转移**：支持工单转移（同部门内），处理记��全程留痕
5. **资源类工单联动**：
   - 资源类工单审核通过后，自动触发云上资源创建/操作流程
   - 工单与对应资源实现一对一绑定
   - 可通过资源查看关联工单记录
   - 可通过工单查看关联资源信息

### 7.4 工单视图
1. **总览**：查看所有工单（根据权限过滤）
2. **我处理的**：仅工单处理人/部门管理员可见，展示待处理和处理中的工单
3. **我创建的**：展示当前用户创建的所有工单

### 7.5 工单权限与查看
1. **查看权限**：
   - 部门负责人可查看本部门所有工单
   - 组内工单处理人可查看本组负责工单
   - 申请人可查看自己创建的工单
2. **筛选功能**：
   - 搜索工单号/标题
   - 按工单类别筛选
   - 按工单状态筛选（待处理、处理中、已完结）
   - 按部门筛选

### 7.6 工单统计
1. **统计维度**：
   - 按部门/组统计
   - 按工单类型统计
   - 按处理人统计
   - 按时间维度统计
2. **统计指标**：
   - 工单数量
   - 处理时长
   - 完结率
3. **统计展示**：工单管理页面顶部展示统计卡片，为运维团队工作考核提供数据支撑

### 7.7 工单处理人配置
1. **配置入口**：部门配置 → 工单处理配置
2. **配置主体**：部门负责人
3. **配置内容**：为部门内成员指定工单处理人角色，支持按工单类型分配不同处理人

## 八、部门配置管理

### 8.1 配置入口
1. **访问权限**：
   - 超级管理员可查看和配置所有部门
   - 部门负责人仅可查看和配置本部门
2. **部门选择**：超级管理员通过下拉选择切换部门，部门负责人直接显示本部门名称

### 8.2 配置模块
1. **主账号配置**：
   - 绑定/解绑阿里云主账号
   - 配置主账号的访问凭证
   - 查看主账号绑定状态
2. **资源操作配置**：
   - 配置各资源类型的申请表单模板
   - 定制表单字段（固定字段、可选字段）
   - 配置字段默认值
3. **资源审批配置**：
   - 配置各资源类型、操作类型的审批流程
   - 指定审批节点和审批人
   - 配置审批规则（如特殊资源需额外审批）
4. **工单处理配置**：
   - 为部门成员分配工单处理人角色
   - 配置工单类型与处理人的对应关系
5. **成员角色分配**（仅超级管理员可见）：
   - 为部门成员分配功能模块角色
   - 查看成员当前角色

## 九、操作审计

### 9.1 审计范围
一期实现核心操作全链路审计，覆盖以下6类操作：
1. **资源操作**：资源的申请、变更、销毁、扩缩容、重启、单独申请/扩容、二次申请等
2. **权限变更**：资源授权/回收、角色分配/移除、权限模板调整、子账号申请/创建等
3. **组织架构变更**：组的创建/解散、人员调组、部门/组负责人变更等
4. **云账号操作**：主账号关联/解绑、子账号申请/创建/回收/销毁等
5. **审核流程操作**：审核通过、驳回、转审、撤回等
6. **工单系统操作**：工单创建、处理、转移、完结、统计等

### 9.2 审计日志字段
每条审计记录包含以下必选字段：
- 操作人
- 操作时间
- 操作类型
- 操作对象（资源ID/用户ID/组ID/项目ID/工单ID）
- 操作前状态
- 操作后状态
- 所属部门
- 来源IP

### 9.3 审计日志可见性
1. **查看权限**：仅管理角色可查看审计日志
   - 超级管理员
   - 部门负责人
   - 运维负责人
   - 工单处理人
2. **数据隔离**：按部门做数据隔离，非管理角色无查看权限
3. **筛选功能**：
   - 按操作类型筛选
   - 按操作人筛选
   - 按时间范围筛选
   - 按操作对象筛选

## 十、一期核心边界与特殊说明

### 10.1 权限拦截
一期仅由CMP平台侧做权限拦截，统一使用部门主账号对接云上资源，暂不实现平台+云上的双层拦截。平台需做好权限校验，避免权限穿透风险。

### 10.2 资源范围
1. 一期仅管控云上资源，云下资源暂不纳入管理范围
2. 仅管控业务实际在用的云上资源，未实际使用的云上资源暂不纳入资源目录

### 10.3 项目与资源包
1. 一期仅做项目，不做用户组
2. 项目为资源的逻辑分组容器，支持批量管理与批量授权
3. 所有资源申请时必须选择一个项目，一个资源仅能绑定一个项目
4. "资源包"（多个资源及权限的打包集合，用于批量授权给用户）为独立概念，与项目互不干扰

### 10.4 权限模板
1. 云上子账号权限模板由平台超级管理员/运维统一预置
2. 按粗粒度打包（如K8s集群管理、ECS操作、MySQL管理、管理员权限等）
3. 一期不开放用户自定义模板
4. 工单处理人、审核人等角色权限模板也由超级管理员统一预置

### 10.5 二次申请
一期考虑云上资源二次申请逻辑（如Kafka Topic、ES索引），在审核流程中新增对应操作类型，并将二次申请纳入工单系统管理，实现流程留痕。

### 10.6 与其他系统联动
1. **与SMDB平台**：
   - CMP平台与SMDB平台为独立系统
   - 一期仅做基础数据同步
   - CMP专注资源申请与权限管理、工单处理
   - SMDB专注元数据与业务视角资源拓扑
   - 边界清晰
2. **与ERP低代码平台**：
   - 深度联动
   - 审核流程、工单审核均基于ERP平台配置
   - 实现流程统一

### 10.7 云上接口适配
1. 平台抽象云上资源核心不变字段（如地域、规格、项目）做统一配置
2. 云上接口新增非核心字段不影响平台现有功能，仅按需迭代补充
3. 云上接口变动时，平台仅同步调整对应资源表单模板，不改动核心架构

### 10.8 模板管理复杂度
资源申请表单模板的配置复杂度由部门运维团队承担，平台侧做好公共字段抽象与复用，确保终端用户申请操作极简（仅需填写3-5个核心字段）。

### 10.9 工单系统实现
一期不复用现有平台工单系统，在CMP平台单独搭建专属工单系统，支持部门负责人配置工单处理人，按部门做数据隔离，适配业务线运维管理需求。

### 10.10 成本核算
一期不做成本核算，成本分析（按部门/组/项目维度的费用分摊、资源利用率统计等）规划至二期建设。

### 10.11 组间资源转移
一期暂不支持组间资源转移功能。

## 十一、功能模块总览

### 11.1 核心功能模块
